IEnumerable vs IQueryable în C#

Două interfețe foarte folosite în LINQ și Entity Framework. Află diferențele și când să le folosești.

1) Ce este IEnumerable<T>?

Definiție scurtă: IEnumerable<T> reprezintă o colecție deja în memorie pe care o poți parcurge element cu element. Filtrarea/transformarea LINQ se face după ce datele au fost aduse în memorie.

IEnumerable este o interfață definită în System.Collections și folosită pentru a parcurge colecții încărcate în memorie: List<T>, Array, etc.

Operațiile LINQ pe IEnumerable sunt executate în memorie după ce toate datele au fost aduse. 

IEnumerable<int> numbers = new List<int> {1, 2, 3, 4, 5};
var result = numbers.Where(n => n > 3);

foreach (var n in result)
    Console.WriteLine(n); // filtrează după ce lista este în memorie
✔ Bun pentru colecții mici și pentru logica simplă aplicată în client.

2) Ce este IQueryable<T>?

Definiție scurtă: IQueryable<T> construiește un arbore de expresii LINQ care este tradus în SQL (sau în limbajul sursei) și rulat pe server. Astfel, aduci în memorie doar datele necesare.

IQueryable este definită în System.Linq și este folosită când datele provin din surse externe (de exemplu baze de date, prin Entity Framework).

Operațiile LINQ pe IQueryable se traduc în SQL (sau alt limbaj) și sunt executate direct pe server. 

IQueryable<Student> students = context.Students.Where(s => s.Age > 18);
var top2 = students.Take(2);

// tradus în SQL: SELECT TOP (2) * FROM Students WHERE Age > 18
✔ Ideal pentru seturi mari de date, deoarece filtrarea se face pe server.

3) Tabel comparativ

Caracteristică IEnumerable IQueryable
Namespace System.Collections System.Linq
Loc de execuție În memorie (client-side) Pe server (server-side)
Surse tipice List, Array Entity Framework, LINQ to SQL
Performanță Mai lent pentru seturi mari Mai eficient pentru volume mari
Execuție întârziată Da Da

4) Când să le folosești?

  • Folosește IEnumerable când lucrezi cu colecții deja în memorie.
  • Folosește IQueryable când vrei să delegi filtrarea și selectarea datelor către baza de date.

5) Concluzie

Pe scurt: IEnumerable lucrează cu date deja încărcate, iar IQueryable construiește expresii care sunt transformate în SQL și executate pe server.

Alegerea corectă depinde de sursa datelor și de volumul lor. În .NET, de multe ori vei combina cele două: IQueryable pentru a aduce datele eficiente din DB, apoi IEnumerable pentru procesări în memorie.
