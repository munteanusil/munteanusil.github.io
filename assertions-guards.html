Guard Clauses & Assertions în .NET

„Garda la ușă” + „alarma din casă”. Împreună fac codul mai sigur și mai ușor de citit.

1) Ce sunt Guard Clauses?

Guard Clause este un pattern care se folosește pentru a valida datele de intrare într-o metodă sau funcție.

📌 Ideea: înainte să rulezi logica principală, verifici că argumentele primite respectă regulile de business sau constrângerile minime. Dacă nu → oprești imediat execuția, de obicei printr-o excepție.

Exemplu din viața reală: la intrarea într-un club, un bodyguard verifică dacă ai bilet valid. Dacă nu, te oprește înainte să intri. Asta face și guard clause cu parametrii.

  • se pun la începutul metodelor (fail-fast);
  • protejează aplicația de date invalide sau neașteptate;
  • reduc codul „nested” (evită if-uri în cascadă).
public void RegisterUser(string email, int age)
{
    if (string.IsNullOrWhiteSpace(email))
        throw new ArgumentException("Email obligatoriu.", nameof(email));

    if (age < 13)
        throw new ArgumentOutOfRangeException(nameof(age), "Minim 13 ani.");

    // logica continuă doar cu date valide
}

2) Ce sunt Assertions?

Assertion este un mecanism folosit în timpul dezvoltării și testării pentru a verifica dacă presupunerile interne ale programului sunt corecte.

📌 Ideea: „Eu, ca programator, știu că în acest punct al codului ceva trebuie să fie adevărat. Dacă nu e → am un bug.”

Exemplu din viața reală: îți verifici portofelul după ce ai plătit la casă. Te aștepți să mai ai cardul acolo. Dacă nu e → ceva nu a mers bine în procesul tău.

  • folosite mai ales în Debug/Testing, nu pentru inputul utilizatorului;
  • dacă pică, semnalează o problemă de implementare;
  • în .NET, se folosesc cu Debug.Assert.
using System.Diagnostics;

public Order MapToOrder(OrderDto dto)
{
    Debug.Assert(dto != null, "DTO validat deja la intrare.");

    var order = new Order(dto.Id, dto.CustomerId);
    Debug.Assert(order.Id != Guid.Empty, "Id nu trebuie să fie gol.");

    return order;
}

3) Împreună, într-un API

[HttpPost("users")]
public IActionResult Post(RegisterUserDto dto)
{
    // Guard: input din exterior
    if (string.IsNullOrWhiteSpace(dto.Email))
        return BadRequest("Email necesar");

    _userService.Register(dto.Email, dto.Age); // aici pot avea Debug.Assert pentru invariante interne
    return Ok();
}

4) Diferența cheie

  • Guard Clauses: pentru date din afară (utilizator, API, fișiere, baze de date).
  • Assertions: pentru invariante interne (logica ta trebuie să respecte anumite reguli).

5) Cum lucrează împreună?

Un proiect matur le folosește pe amândouă:

  • Guard Clauses filtrează gunoiul din exterior.
  • Assertions confirmă că logica internă a codului e solidă.

Împreună oferă un dublu nivel de protecție:

  • Evită date invalide.
  • Evită bug-uri ascunse.

Regula ușoară

  • Input din afară (user, API, fișiere) → Guard clause.
  • Invariante interne („nu ar trebui să se întâmple”) → Assertion.

Greșeli frecvente

  • Folosirea Assert pentru input de la utilizator (utilizatorul nu vede nimic în producție).
  • Mesaje vagi („Invalid data”) – spune clar ce e greșit.
  • Validări duplicate la fiecare strat fără motiv.
