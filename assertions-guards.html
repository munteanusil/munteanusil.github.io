Guard Clauses & Assertions în .NET

„Garda la ușă” + „alarma din casă”. Împreună fac codul mai sigur și mai ușor de citit.

1) Ce sunt Guard Clauses?

Guard Clause este un pattern care se folosește pentru a valida datele de intrare într-o metodă sau funcție.

📌 Ideea: înainte să rulezi logica principală, verifici că argumentele primite respectă regulile de business sau constrângerile minime. Dacă nu → oprești imediat execuția, de obicei printr-o excepție.

Exemplu din viața reală: la intrarea într-un club, un bodyguard verifică dacă ai bilet valid. Dacă nu, te oprește înainte să intri. Asta face și guard clause cu parametrii.

  • se pun la începutul metodelor (fail-fast);
  • protejează aplicația de date invalide sau neașteptate;
  • reduc codul „nested” (evită if-uri în cascadă).
public void RegisterUser(string email, int age)
{
    if (string.IsNullOrWhiteSpace(email))
        throw new ArgumentException("Email obligatoriu.", nameof(email));

    if (age < 13)
        throw new ArgumentOutOfRangeException(nameof(age), "Minim 13 ani.");

    // logica continuă doar cu date valide
}

2) Assertion Methods (ca Guard Clauses reutilizabile)

Assertion Methods sunt metode statice de validare pe care le scrii o singură dată și apoi le folosești ca „mini-guard clauses” în toată aplicația. Sunt parte din Defensive Programming: reduc codul duplicat, mențin regulile consecvente și păstrează logica de business curată.

Pe scurt: un guard clause extras într-o metodă reutilizabilă = assertion method.

Exemplu 1 — Valoare pozitivă

// Assertion Method
public static class Ensure
{
    public static void Positive(decimal number, string message, string? paramName = null)
    {
        if (number <= 0) throw new ArgumentException(message, paramName);
    }
}

// Folosire ca Guard Clause
public void CreateInvoice(decimal amount)
{
    Ensure.Positive(amount, "Valoarea facturii trebuie să fie > 0.", nameof(amount));
    Console.WriteLine($"Factură creată cu {amount} lei.");
}

Exemplu 2 — Email valid (simplu)

public static class Ensure
{
    public static void ValidEmail(string email, string message, string? paramName = null)
    {
        if (string.IsNullOrWhiteSpace(email) || !email.Contains("@"))
            throw new ArgumentException(message, paramName);
    }
}

// Folosire
public void RegisterUser(string email)
{
    Ensure.ValidEmail(email, "Email-ul introdus nu este valid.", nameof(email));
    Console.WriteLine("User înregistrat cu succes!");
}

Exemplu 3 — Nu null / nu gol (helper clasic)

public static class Ensure
{
    public static string NotNullOrWhiteSpace(string? value, string message, string? paramName = null)
    {
        if (string.IsNullOrWhiteSpace(value)) throw new ArgumentException(message, paramName);
        return value!;
    }
}

// Folosire + chaining
public void CreateProject(string name, decimal budget)
{
    var safeName = Ensure.NotNullOrWhiteSpace(name, "Numele proiectului este obligatoriu.", nameof(name));
    Ensure.Positive(budget, "Bugetul trebuie să fie > 0.", nameof(budget));

    // logica continuă doar cu date valide
    Console.WriteLine($"Proiect '{safeName}' cu buget {budget} lei.");
}

De ce să le folosești

  • Mai curat: extragi if-urile repetitive într-un singur loc.
  • Reutilizabil: aceeași regulă se aplică în multe servicii/straturi.
  • Ușor de schimbat: dacă regula evoluează (ex. email cu regex), modifici o singură metodă.
  • Fail-fast consecvent: arunci excepții clare (ex. ArgumentException / ArgumentOutOfRangeException).
Sfat practic: păstrează metodele în clasa statică Ensure (sau Guard) dintr-un proiect comun. Astfel, toate serviciile folosesc aceleași reguli.
